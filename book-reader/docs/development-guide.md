# Руководство по быстрой разработке

Цель этого файла — помогать вносить маленькие правки быстро, а большие изменения делать предсказуемо.

## Источники правды

| Что меняется | Где смотреть и править |
| --- | --- |
| позиционирование, голос, утверждённые тексты | `docs/project-base.md` |
| названия глав и статус страниц | `docs/book-contents.md` |
| страницы, главы, ссылки, цена, доступ | `src/reader-app.jsx` |
| визуал и адаптивность | `src/reader.css` |
| Supabase-клиент | `src/supabase-client.js` |
| синхронизация прогресса | `src/use-reading-sync.js` |
| схема базы и RLS | `supabase/migrations/` |
| релизные ограничения | `docs/implementation-status.md`, `RELEASE_CHECKLIST.md` |

Не дублировать данные в новых местах без необходимости.

## Быстрый цикл

```bash
npm install
npm run dev
```

После изменений:

```bash
npm run build
```

Сборка одновременно проверяет отсутствующие и неиспользуемые страницы, видео и шрифты.

## Типовые изменения

### Заменить страницу книги

1. Положить WebP в `public/pages/` с существующим системным именем.
2. Для новой страницы добавить запись в `PAGES`.
3. Проверить номер, `kind`, главу и внешнюю ссылку, если это музыкальная страница.
4. Обновить диапазон и количество страниц в `CHAPTERS`.
5. Обновить `docs/book-contents.md` и `docs/implementation-status.md`.
6. Выполнить `npm run build` и пролистать соседние развороты.

Рекомендуемый исходный размер страницы: `1440 × 2080`, единый для всего ридера.

### Добавить музыкальную ссылку

Добавить `href` к нужной записи `PAGES`. Ссылка должна открываться в новой вкладке через существующий компонент `BookPage`. После правки проверить клик мышью и клавиатурой.

### Изменить главную или оглавление

Сначала найти существующий компонент и CSS-селектор. Не создавать параллельную версию экрана. Декоративные элементы держать вне слоя текста и проверять, что они не перекрывают контент на промежуточных ширинах.

Тетрадная сетка оглавления использует одну переменную `--toc-row-height` в
`.toc__paper`. Она одновременно задаёт шаг фоновых линий и высоту строк глав.
При изменении плотности оглавления менять эту переменную, а не разносить
независимые значения по строкам и фону.

### Изменить доступ

Проверить вместе:

- `PREVIEW_ALL_CHAPTERS`;
- `LAST_FREE_PAGE`;
- `FIRST_PAID_PAGE`;
- вычисление `hasFullAccess`;
- `book_entitlements`;
- поведение оглавления и модального окна.

Визуальная задача не должна незаметно менять эти значения.

## Обязательная визуальная проверка

Минимальный набор размеров:

- desktop: `1440 × 900`;
- узкий desktop/tablet: `1000 × 800`;
- tablet portrait: `768 × 1024`;
- mobile: `390 × 844`.

На каждом размере проверить:

- нет горизонтального скролла;
- обложка и текст не перекрываются;
- декоративные SVG не попадают на текст и навигацию;
- разворот не наезжает сам на себя;
- страницы полностью помещаются по высоте либо корректно прокручиваются;
- стрелки навигации доступны и не закрывают содержимое;
- оглавление, покупка и возврат работают;
- внешние ссылки открываются в новой вкладке.

## Производительность

- Использовать WebP для страниц и оптимизированные SVG для doodle.
- Не подключать библиотеку ради одной анимации или иконки.
- Не загружать одновременно изображения, которые не нужны в текущем развороте.
- Сохранять фиксированное соотношение сторон страниц, чтобы интерфейс не прыгал при загрузке.
- Удалять неиспользуемые ассеты; `npm run build` должен оставаться зелёным.
- Для декоративных анимаций использовать `transform` и `opacity`, не свойства, вызывающие постоянный перерасчёт layout.
- Уважать `prefers-reduced-motion`.

## Definition of Done

Правка завершена, когда:

1. Выполнено только запрошенное изменение без побочных правок.
2. `npm run build` проходит.
3. Проверены затронутые desktop и mobile состояния.
4. Не сломаны чтение, ссылки, доступ и прогресс.
5. При изменении продукта, контента или структуры обновлена документация.

## Перед релизом

Главная опасность текущей версии — временный `PREVIEW_ALL_CHAPTERS = true`. Нельзя считать production-релиз готовым, пока флаг не выключен и реальная оплата не выдаёт подтверждённое право доступа.
