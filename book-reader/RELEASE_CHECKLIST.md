# Чек-лист релиза

## Уже готово

- Проект отделён от `Personal Cosmic Book`.
- Локальная и production-сборка работают через Vite.
- React и JSX собираются заранее, без runtime-Babel и CDN-скриптов.
- Все страницы книги и вступительное видео проверяются перед сборкой.
- Настройки Vercel хранятся в репозитории.
- Supabase подключён отдельным клиентом только внутри `book-reader`.
- Прогресс чтения сохраняется локально и синхронизируется после входа по email.
- Таблицы `reading_progress` и `book_entitlements` созданы с RLS-политиками.
- Старый адрес `/Reader.html` перенаправляется на `/`.
- Desktop, tablet и mobile не имеют горизонтального скролла.

## До публичного запуска

- Выключить временный режим `PREVIEW_ALL_CHAPTERS` в `src/reader-app.jsx`.
- Подключить настоящую оплату вместо демо-формы.
- Создавать `book_entitlements` из доверенного webhook после успешной оплаты.
- Проверить, что первая глава доступна без покупки, а остальные открываются только при наличии права доступа.
- Добавить production-домен в Supabase Authentication → Redirect URLs.
- Добавить политику конфиденциальности, оферту и контакты продавца.
- Подключить финальный домен и проверить HTTPS.
- Настроить аналитику только после решения по согласию на cookies.
- Провести финальную проверку текстов, автора, цены и состава книги.
- Проверить все внешние ссылки на песни и открытие в новой вкладке.
- Проверить прогресс чтения на двух устройствах под одним пользователем.
- Проверить отсутствие горизонтального скролла и наложений на desktop, tablet и mobile.
- Проверить размер production-сборки и загрузку страниц на медленном соединении.

## Деплой

1. Импортировать репозиторий в Vercel.
2. Оставить корень репозитория как Root Directory.
3. Проверить автоматически найденный preset `Vite`.
4. Запустить Preview Deployment.
5. Выполнить `npm run build` локально и пройти визуальную проверку.
6. После приёмки перевести deployment в Production.
