# Personal Cosmic Book

Сервис генерации персональных digital-книг: полная книга для себя и подарочная книга для близкого человека.

## Пайплайн генерации

1. **Расчёт (без LLM)** — `api/chart.py`: натальная карта через `kerykeion`, нумерология и саджу. Координаты города резолвятся локальной таблицей + геокодером; endpoint деградирует мягко, если астрология недоступна.
2. **Назначение темы (без LLM)** — `api/theme.py`: детерминированный модуль `astrology-four-elements-v1`. По стихиям знаков (Солнце 3, Луна 3, Асцендент 2 при известном времени, Меркурий/Венера/Марс 2, Юпитер/Сатурн 1) выбирается одна из четырёх тем оформления. Саджу, нумерология и матрица судьбы для выбора темы не используются.
3. **Генерация текста (LLM)** — `api/book.py`: книга из **ровно 10 разделов** (`book-structure-10-sections.md`). Текст пишет DeepSeek (`DEEPSEEK_API_KEY`); без ключа собирается детерминированный шаблон. В LLM не передаются тема, токены и любые данные оформления.
4. **Рендер (без LLM)** — `public/book.html`: один layout, **четыре фиксированные стихийные темы** (`earth | water | air | fire`) из неизменяемых токенов (`design-system-elements-v2.md`). Меняются только токены, символ стихии и паттерн; структура, сетка и типографика одинаковы.

## Дизайн-система: 4 стихии

`earth · Земля`, `water · Вода`, `air · Воздух`, `fire · Огонь`. Пользователь выбирает тему в форме заказа (5 карточек: «Подобрать по карте» + 4 стихии) или получает авто-назначение по карте. Тема — это только оформление, она не влияет на расчёты и текст.

Legacy-темы мигрируют: `cosmic_night → water`, `solar_dawn → fire`, `lunar_mist → air`.

## Локальный запуск

```bash
pnpm install
pnpm dev            # витрина (лендинг отдаётся на / через rewrite)
```

Тесты Python:

```bash
python3 -m pip install -r requirements.txt   # kerykeion для полной астрологии
python3 -m pytest tests/                      # chart, theme assignment, структура книги
```

## Переменные окружения

См. `.env.example`. Ключевые: `DEEPSEEK_API_KEY` (генерация текста), `YOOKASSA_SHOP_ID` / `YOOKASSA_SECRET_KEY` (оплата), Supabase-ключи.

## Спецификация

Продуктовая и техническая спецификация — в `prompts/` (версия v5): структура 10 разделов, дизайн-система, контракты входных данных и renderer, стадии генерации, план оценки.
