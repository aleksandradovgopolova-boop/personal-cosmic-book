# Personal Cosmic Book — prompts v5

## Что зафиксировано в этой версии

1. Основная книга содержит **ровно 10 смысловых разделов**.
2. Продукт использует **одну дизайн-систему и четыре фиксированные темы**, связанные с классическими стихиями:
   - `earth` — «Земля · Корни»;
   - `water` — «Вода · Глубина»;
   - `air` — «Воздух · Дыхание»;
   - `fire` — «Огонь · Искра».
3. Пользователь может выбрать оформление самостоятельно или оставить `auto`.
4. Автоматический выбор темы выполняется **кодом**, а не LLM, на основе рассчитанного баланса четырёх стихий западной астрологической карты.
5. Саджу не используется для выбора темы: его пятиэлементная система не должна искусственно сводиться к четырём классическим стихиям.
6. LLM отвечает только за смысл книги. Она не выбирает палитру, CSS, шрифты, layout, символ и визуальную тему.
7. Уникальный дизайн каждой книги отменён. Персональность создаётся текстом, именем, названием, выбранной стихией и детерминированной графической подписью.

## Архитектура книги

Старые 19 глав не склеиваются механически. Новая драматургия строится вокруг пути читателя:

```text
главная тема
→ внутренний мир
→ сильные стороны
→ внутренние противоречия
→ любовь и близость
→ реализация, дело и деньги
→ проявление в мире
→ повторяющиеся сценарии
→ персональная формула
→ следующий шаг
```

Астрология, нумерология, матрица судьбы и Саджу не являются отдельными главами. Они используются как evidence и объединяются в целостном синтезе.

## Файлы

1. `book-structure-10-sections.md` — редакторская архитектура десяти разделов.
2. `input-data-contract-v3.md` — нормализованный контракт данных, presentation preferences и результат детерминированного назначения темы.
3. `stage2-pattern-synthesis-v2.md` — синтез паттернов со ссылками на evidence ID.
4. `stage3-book-blueprint.md` — blueprint ровно из десяти разделов.
5. `stage4-section-generation.md` — генерация одного раздела за вызов в JSON, без HTML и дизайна.
6. `renderer-contract-v2.md` — детерминированная HTML/PDF-сборка и правила четырёх тем.
7. `design-system-elements-v2.md` — полная система «Земля / Вода / Воздух / Огонь».
8. `quality-validation-v4.md` — программная, смысловая и визуальная QA.
9. `stage5-addons-v2.md` — прогноз и совместимость через тот же безопасный пайплайн.
10. `model-routing.md` — рекомендуемый роутинг DeepSeek/Kimi.
11. `evaluation-plan.md` — сравнительный тест моделей и четырёх тем.
12. `claude-implementation-prompt.md` — готовая задача для Claude по внедрению изменений в репозиторий.

## Рекомендуемый пайплайн

```text
локальные расчёты
  → нормализация + evidence catalog
  → детерминированный element profile
  → синтез паттернов
  → blueprint ровно из 10 разделов
  → 10 последовательных генераций разделов JSON
  → программная QA
  → semantic QA и точечные исправления
  → выбор одной из 4 тем кодом
  → HTML/PDF renderer
  → visual и print validation
```

## Главный принцип

**LLM пишет смысл. Код отвечает за вычисления, структуру, схемы, выбор темы, дизайн, сборку и проверки.**

Тема оформления — визуальное настроение книги, а не диагноз, «истинная стихия личности» или новый интерпретационный вывод.
