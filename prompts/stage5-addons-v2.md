# Этап 5: дополнения v2

## Общий принцип

Каждое дополнение проходит собственные deterministic calculations, synthesis, blueprint, section generation и QA. Оно не должно вытягивать значения из основной книги или пересчитывать их моделью.

## Оформление дополнений

1. По умолчанию прогноз и совместимость наследуют `resolved_theme` основной книги.
2. Они используют ту же дизайн-систему и те же tokens.
3. Не создавать отдельные палитры для прогнозов и совместимости.
4. Пользователь может изменить тему только через общий selector; изменение не влияет на расчёты и текст.
5. Для совместимости пары не вычислять «общую стихию дизайна» через LLM.
6. Если дополнение создаётся без основной книги, theme assignment выполняется тем же серверным модулем.

# Прогноз на год

## Дополнительный вход

```json
{
  "target_period": {
    "start": "2026-01-01",
    "end": "2026-12-31",
    "timezone": "Europe/Moscow"
  },
  "transit_evidence_catalog": [],
  "natal_pattern_ids": [],
  "user_focus": []
}
```

## Правила

1. Без `transit_evidence_catalog` прогноз не генерируется.
2. Не повторять общий портрет личности.
3. Разделять:
   - расчётный период;
   - интерпретационную гипотезу;
   - возможный фокус;
   - практическое действие.
4. Не утверждать, что событие произойдёт.
5. Не давать точных обещаний брака, беременности, болезни, увольнения, богатства, потерь или смерти.
6. Для каждого периода указывать evidence refs.
7. Использовать формулировки «период может усилить», «полезно обратить внимание», «вероятной темой становится».

## Структура

1. Главный фокус года
2. Карта периодов
3. Работа и самореализация
4. Отношения и близость
5. Деньги и ресурсы
6. Внутренняя устойчивость
7. План наблюдений и действий

# Совместимость

## Дополнительный вход

```json
{
  "person_a": {},
  "person_b": {},
  "relationship_context": {
    "relationship_type": "romantic | friendship | family | work",
    "duration": null,
    "main_request": "",
    "address_to": "person_a | person_b | couple"
  },
  "synastry_evidence_catalog": []
}
```

## Правила

1. Сначала каждый человек проходит этапы 1–2 отдельно.
2. Совместимость описывает взаимодействие, а не оценивает людей как хороших или плохих.
3. Не делать выводов о неизбежности союза или расставания.
4. Не оправдывать насилие, контроль, ревность и нарушение границ символическими показателями.
5. Не выдавать отсутствие совместимости за приговор.
6. Различать:
   - индивидуальные паттерны A;
   - индивидуальные паттерны B;
   - расчётные парные сигналы;
   - пользовательский контекст.
7. Каждый вывод должен иметь ссылки на соответствующие pattern/evidence IDs.

## Структура

1. Главная динамика пары
2. Где люди усиливают друг друга
3. Как они проявляют близость
4. Где могут не слышать друг друга
5. Быт, деньги и решения
6. Границы и способы восстановления контакта
7. Практические договорённости
