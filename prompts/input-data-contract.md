# Этап 1: Контракт входных данных

## Назначение

Этап 1 не использует Claude. Он считает карту и символические системы локально, а затем передает готовый JSON в этап 2.

## Источники данных

- `api/chart.py`: натальная карта через `kerykeion`
- Python-расчет: нумерология
- Python-расчет: матрица судьбы
- Python-расчет: саджу

## Правила

1. Промпты не считают значения сами.
2. Все вычисления должны быть завершены до передачи JSON в этап 2.
3. Если время рождения неизвестно, карта помечается как менее точная, а дома/углы используются осторожно.
4. `chart_data` кэшируется по `hash(дата + время + место)`.

## Нумерология

- Число жизненного пути: сумма цифр даты рождения до однозначного числа.
- Мастер-числа `11`, `22`, `33` не редуцируются.
- Число имени считается по русской таблице Пифагора.

## Матрица судьбы

- Значения приводятся к диапазону `1-22`.
- Если сумма больше `22`, редуцируем сложением цифр.
- `0` трактуется как `22`.
